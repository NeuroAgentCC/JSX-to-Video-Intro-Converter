# JSX to Video Intro Converter

Доступно на: [English](README.md) | **Русский**

Конвертирует JSX-анимации логотипов (экспортированные дизайн-инструментами) в MP4-видео — Full HD, 2K, 4K или 8K, горизонтально или вертикально. Достаточно кинуть ZIP-архив и запустить.

## Демо

Конвертирует JSX компоненты в видео — два способа запуска: через диалог Claude Code или терминал.

[видео]

## Способ 1 — через Claude Code (рекомендуется)

Если у тебя установлен [Claude Code](https://claude.ai/code) — открой папку проекта в нём и просто напиши **"делай"**. Claude сам спросит нужный формат и длительность прямо в диалоге, после чего запустит конвертацию.

Claude отвечает на том языке, на котором ты пишешь — поддерживаются русский и английский.

Поддерживаемые форматы: Full HD (1920×1080), 2K, 4K, 8K. Ориентация (горизонталь/вертикаль) определяется автоматически для каждого файла.

## Способ 2 — через терминал

1. Положи ZIP-архив с анимацией в папку **`Input`**
2. Запусти в терминале из папки проекта:
   ```bash
   python3 convert.py
   ```
3. Скрипт задаст вопросы про формат и длительность — ответь на них
4. Готовые MP4 появятся в папке **`Output/<имя архива>/`**

Если в архиве несколько версий анимации — каждая конвертируется отдельно. Исходный ZIP автоматически перемещается в папку с результатами. Ориентация (горизонталь/вертикаль) определяется автоматически для каждого файла.

## Подготовка архива (для пользователей Claude Design)

Скрипт работает с любым ZIP-архивом — он найдёт и сконвертирует все HTML/JSX-файлы которые обнаружит внутри. Но если ты генерируешь анимации в Claude Design, отправь этот промт **перед скачиванием архива** — тогда структура будет правильной: в конвертацию попадут только финальные варианты, а превью-страницы и объединённые файлы останутся отдельно.

```
Теперь для каждой версии интро сделай отдельный JSX-файл, чтобы мы могли корректно сконвертировать каждую версию в видео через наш скрипт.

Важно: не размещай все интро на одной общей HTML-странице или в одном общем JSX-файле. Для каждого отдельного варианта интро должен быть создан свой отдельный JSX-файл.

Так как результат мы скачиваем одним ZIP-архивом, обязательно раздели файлы по папкам внутри архива.

Финальные JSX-файлы для конвертации положи только в папку:

for_conversion/

В папке for_conversion/ должны лежать только чистые финальные JSX-файлы, где каждый файл содержит ровно один интро-вариант.

Старые объединённые файлы, HTML-страницы с меню, превью-страницы, файлы со всеми интро сразу и любые вспомогательные материалы положи отдельно в папку:

not_for_conversion/

Папка not_for_conversion/ нужна только для архива или просмотра. Наш скрипт не должен использовать файлы из этой папки для конвертации.

Названия файлов в папке for_conversion/ не должны быть абстрактными вроде Intro_01.jsx, Intro_02.jsx и так далее. Название каждого JSX-файла должно соответствовать названию нашего проекта и формату видео.

Например, если проект называется ProjectName, то структура должна быть такой:

* for_conversion/ProjectName_16x9_Variant_01.jsx
* for_conversion/ProjectName_16x9_Variant_02.jsx
* for_conversion/ProjectName_9x16_Variant_01.jsx
* for_conversion/ProjectName_9x16_Variant_02.jsx

Если формат видео 16:9, это обязательно должно быть указано в названии файла как 16x9. Если формат видео 9:16, это обязательно должно быть указано в названии файла как 9x16.

Каждый JSX-файл в папке for_conversion/ должен содержать только одну конкретную версию интро. Не объединяй несколько вариантов в один файл.

Если для удобного просмотра ты хочешь сделать общую HTML-страницу с меню, превью или переключением между интро — это можно сделать отдельно, но только в папке not_for_conversion/.

Меню, кнопки, навигация, список вариантов и любые элементы интерфейса не должны попадать внутрь JSX-файлов из папки for_conversion/.

В самих JSX-файлах из папки for_conversion/ должен быть только чистый интро-вариант, без меню, без кнопок, без лишней обвязки и без элементов, которые потом придётся вырезать вручную.

Сами варианты интро оставь такими же, как уже были сгенерированы ранее, но разнеси их по отдельным JSX-файлам и положи только в папку for_conversion/.

Важно: в папке for_conversion/ не должно быть старых общих JSX-файлов, HTML-страниц, превью-страниц или файлов, где несколько интро находятся вместе. Там должны быть только файлы, которые нужно конвертировать в видео.

Технические требования к каждому JSX-файлу в папке for_conversion/:

Каждый JSX-файл должен быть полностью самодостаточным и не зависеть ни от каких других файлов из архива. Конкретно:

1. Никаких импортов и никаких ссылок на внешние JSX, JS или CSS файлы. Всё необходимое — только внутри самого файла.

2. Только window.React и window.ReactDOM как внешние зависимости (React 18). Никаких других глобальных переменных или библиотек.

3. Никакого JSX-синтаксиса, требующего Babel. Только React.createElement(). Файл должен работать как обычный JS без транспиляции.

4. В начале файла обязательно объявить три константы:
   - FRAME_W — ширина кадра в пикселях (например, const FRAME_W = 1920)
   - FRAME_H — высота кадра в пикселях (например, const FRAME_H = 1080)
   - DURATION — длительность анимации в секундах (например, const DURATION = 5)

5. При загрузке страницы файл должен автоматически монтироваться в элемент <div id="root"> и сразу начинать воспроизведение анимации без какого-либо взаимодействия с пользователем.

6. Все ассеты (изображения, логотипы, иконки) должны быть встроены прямо в JSX-файл в формате base64. Никаких ссылок на файлы из папок uploads/, assets/ и т.п.
```

## Требования

- Python 3.9+
- Интернет при первом запуске (установка зависимостей + загрузка React с CDN при рендере)

При первом запуске скрипт сам проверит наличие всего необходимого и предложит установить, если чего-то не хватает.

## Примечание

Скрипт сделан для собственной задачи — конвертировать JSX-анимации логотипа в видео. Решили поделиться, вдруг кому пригодится.

Запуск через терминал не зависит ни от каких внешних инструментов и работает всегда.
